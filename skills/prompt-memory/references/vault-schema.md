# Vault JSON Schema

The vault file `.promptcraft/prompt_vault.json` stores an immutable append-only log of
prompt checkpoints. Each entry represents one version of a prompt for a given task.

## Top-Level Structure

```json
{
  "version": "1",
  "entries": [...]
}
```

## Entry Schema

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `id` | string (UUID) | auto | Unique entry identifier, generated by checkpoint.py. |
| `task_id` | string | yes | Kebab-case task identifier (e.g. `smart-contract-audit`). Groups versions together. |
| `version_tag` | string (v1, v2, ...) | auto | Incremented version number, auto-generated based on existing versions for this task_id. |
| `is_active` | boolean | auto | Only one version per task_id is `true`. hydrate.py returns the active version by default. |
| `parent_version` | string or null | auto | Points to the previous version_tag. null for v1. Forms a linked version chain. |
| `timestamp` | ISO 8601 | auto | UTC timestamp of when the entry was saved. |
| `skill_used` | string | no | The technique used (e.g. `zero-shot`, `tree-of-thought`). |
| `user_intent` | string | yes | The user's original task goal in their own words. |
| `hard_constraints` | array of strings | no | Non-negotiable constraints derived during prompt engineering. |
| `key_decisions` | array of strings | no | Key technique/routing decisions made during prompt construction. |
| `generated_prompt` | string | no | **The complete generated prompt text**, stored in full without truncation. Use `hydrate.py --full` to retrieve it alongside search results. |
| `generated_prompt_preview` | string (≤200 chars) | auto | Auto-generated truncated preview of `generated_prompt`. Included in `--full` results for quick identification. |
| `execution_feedback` | string | no | User feedback on how well the prompt performed (e.g. "F1=94%"). |
| `tags` | array of strings | no | Freeform tags for categorization and semantic search. |

## Version Lifecycle

```
v1 (is_active: false, parent: null)
  └─ v2 (is_active: true, parent: v1)
       └─ v3 (next save, will become active)

Rollback: hydrate.py --rollback-to v1 → v1 becomes active, v2 becomes inactive.
```

## hydrate.py Scoring Weights

When performing semantic search, the following fields are scored with keyword overlap
(Jaccard similarity) against the query:

| Field | Weight |
|-------|--------|
| `user_intent` | ×2.0 |
| `tags` | ×2.0 |
| `hard_constraints` | ×1.0 |
| `key_decisions` | ×1.0 |
| `execution_feedback` | ×0.5 |

`generated_prompt` and `generated_prompt_preview` are NOT included in default search results
to keep context injection under ~500 tokens. Use `hydrate.py --query "..." --full` to include
the complete prompt text when you need to reuse a previously saved prompt.
