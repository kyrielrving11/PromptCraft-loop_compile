# Zero-Shot Prompting

## Purpose

适合目标清晰、无需示例或复杂推理的直接提示词生成。用于简单的代码解释、格式化、修改变量名、增删注释、调整代码风格等低负载、高连续性任务。

## Input Requirements

- **用户任务**：说明希望模型完成什么任务。
- **目标输入**：待处理的代码、文本或数据。
- **输出格式**：说明最终答案的结构、字段、长度、语言或代码格式。
- **可选信息**：角色设定、约束条件、目标用户或使用场景。

零样本提示不插入任务级输入/输出示例；如果需要示例引导，应改用少样本提示。

## Recommended JSON Input

```json
{
  "task": "将文章总结为 3 个关键要点",
  "role": "你是一名专业内容摘要员",
  "target_input": "{用户输入的文章内容}",
  "output_format": "使用 Markdown 列表，只输出 3 条，每条一句话。",
  "format_example": "- 要点 1\n- 要点 2\n- 要点 3",
  "constraints": ["不要添加额外解释", "输出语言为中文"]
}
```

## Method Steps

1. 直接围绕用户任务、输出格式和约束组织最终 Prompt。
2. 确保 Prompt 包含清晰的：角色设定（Role）、具体任务描述（Task）、输出格式要求（Output Format）。
3. 避免加入不必要的推理框架、阶段历史或示例——保持轻量。
4. 一次只给一条指令，不要在一个 Prompt 中混合多个不相关任务。

## Design Rules

- 保持角色、任务指令、上下文和输出格式明确。
- 零样本提示不插入任务级输入/输出示例；如果需要示例引导，应改用少样本提示。
- 可以提供“示例输出格式”或格式骨架，用于约束结构，但不要把它写成 few-shot 示例。
- 输出格式应可验证，如字段名、顺序、句数、列表数量、Markdown 样式或 JSON 结构。
- 使用 `###`、三引号 `"""` 或代码块清晰分隔指令和待处理内容。
- `{用户输入的文本}` 或 `{text input here}` 必须替换为实际用户文本，或作为待用户后续填入的占位符保留。
- 对于复杂推理任务，可考虑 CoT 技术；对于已有示例的任务，可考虑少样本提示。

## When to Use

- 任务简单直接，不需要多步推理。
- 用户输入已经清晰完整，只需整理格式。
- 上下文连续性高（修改已有代码的局部细节）。
- 常见场景：代码格式化、变量重命名、添加注释、简单 CRUD、配置文件修改。

## Example Use Cases

- "把这段代码的变量名从 `x` 改为 `userCount`"
- "给这个函数添加 JSDoc 注释"
- "把这个 JSON 格式化为缩进 2 空格"
- "写一个简单的 hello world 页面"

## Prompt Output Template

当你用 Zero-Shot 技术生成提示词时，按以下 **7 节骨架**输出（第 5 节约略——Zero-Shot 不需要示例或推理框架）。

### 章节骨架

| # | 章节 | 必需 | 说明 |
|---|------|------|------|
| 1 | 角色 | ✓ | 简明角色设定，如"专业前端工程师，精通 React" |
| 2 | 任务 | ✓ | 一句话 |
| 3 | 输入 | ✓ | 目标代码/数据/文件 |
| 4 | 输出格式 | ✓ | 具体产出格式说明，可用"格式示例"骨架（如 `{ "field": "value" }`），但不要写成 few-shot 示例 |
| 5 | — | 省略 | Zero-Shot 不插入示例或推理框架 |
| 6 | 具体实现要求 | ✓ | 简洁的要点列表 |
| 7 | 硬约束 | ✓ | ≤5 条最关键的规则 |
| 8 | 生成要求 | ✓ | 简短验收标准 |

### 格式说明

- 整体保持轻量，总长度通常 ≤ 100 行
- 不加入任何 "示例 1/示例 2" 或 "推理步骤" 类章节
- 输出格式可给出格式骨架（如 JSON 结构），但不能写成带具体值的 few-shot 示例
- 一次只给一条指令，不混合多个不相关任务
- 如果用户提供了示例，应改用 few-shot，不要强行压缩进 zero-shot

### 质量自检

- [ ] 不含 "示例" 或 "Few-Shot" 章节
- [ ] 不含推理框架（那是 CoT 的事）
- [ ] 不含多分支探索（那是 ToT 的事）
- [ ] 输出格式是骨架（字段名），不是带具体值的示例
- [ ] 总长度 ≤ 100 行

## Notes

- 不要给简单任务添加过度复杂的推理框架。
- 如果任务涉及安全、加密或并发，考虑升级到 tree-of-thought 或 step-back。
- 零样本提示是所有其他技术的 Prompt 结构骨架——其他技术（CoT、Step-Back、Least-to-Most）在其基础上叠加特定的推理或分解结构。
